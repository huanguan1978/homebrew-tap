# Terminology Guide / 术语对照指南

This guide keeps wording consistent between the English and Chinese README files.

## Canonical Terms

| Concept | English (Use This) | 中文（建议统一用法） | Avoid / 备注 |
| --- | --- | --- | --- |
| Product principle | privacy first | 隐私优先 | Avoid mixed forms like privacy-oriented / 注重隐私 in headings |
| Product principle | no forced sign-up | 免强制注册 | Keep this wording stable across pages |
| Architecture style | local-first | 本地优先 | Do not mix with offline-first unless it is actually offline-only |
| Architecture style | offline-first | 离线优先 | Use for tools that can run without network by default |
| Work pattern | workflow | 工作流 | Prefer workflow over process in product descriptions |
| Work pattern | batch workflows | 批处理工作流 | Avoid mixing with 批量任务 unless context specifically means scheduled jobs |
| Security product type | secure vault | 安全资料库 | 可在上下文补充为“安全文件工作台” |
| Security capability | file encryption | 文件加密 | Keep it concise; avoid overly technical terms in summary tables |
| UX role | visual companion | 可视化搭档 | Useful for FileShows + ft relationship |
| Connectivity model | on-demand connectivity | 按需联网 | Use when network is required only for specific operations |
| Learning style | learning by doing | 边用边学 | Keeps Gai messaging practical and user-friendly |
| Automation context | terminal workflows | 终端工作流 | Use when describing CLI automation scenarios |

## Usage Notes

- In README.md, prefer the exact English terms from the table above.
- In README_zh.md, prefer the corresponding Chinese terms from the table above.
- If a new concept appears, add it here first, then update both README files.
